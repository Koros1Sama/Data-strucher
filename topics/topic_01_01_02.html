PL/SQL Block Structure

هيكل كتلة PL/SQL

🧱 البنية الأساسية | Basic Structure

تتكون أي كتلة Block في PL/SQL من ثلاثة أقسام: 

DECLARE    -- (اختياري Optional)
    -- المتغيرات، المؤشرات، الاستثناءات المعرّفة
    -- Variables, cursors, user-defined exceptions
BEGIN       -- (إجباري Mandatory)
    -- جمل SQL
    -- جمل PL/SQL
EXCEPTION   -- (اختياري Optional)
    -- الإجراءات عند حدوث الأخطاء
    -- Actions to perform when errors occur
END;        -- (إجباري Mandatory)

📋 تفصيل الأقسام الثلاثة

القسم الكلمة المفتاحية إجباري؟ الوظيفة
Declarative DECLARE ❌ اختياري تعريف المتغيرات، المؤشرات (Cursors)، الاستثناءات المعرّفة من المستخدم
Executable BEGIN...END; ✅ إجباري جمل SQL وجمل PL/SQL — الكود الفعلي
Exception Handling EXCEPTION ❌ اختياري معالجة الأخطاء التي تحدث أثناء التنفيذ

💻 مثال عملي | Example

SET SERVEROUTPUT ON
DECLARE
    -- قسم التعريف: تعريف المتغيرات
    v_name  VARCHAR2(50) := 'Ahmed';
    v_grade CHAR(1)      := 'A';
BEGIN
    -- قسم التنفيذ: الكود الفعلي
    DBMS_OUTPUT.PUT_LINE('Student: ' || v_name);
    DBMS_OUTPUT.PUT_LINE('Grade: ' || v_grade);
EXCEPTION
    -- قسم الأخطاء: معالجة الاستثناءات
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('An error occurred!');
END;
/

📝 نقاط هامة | Key Takeaways

  • القسم الوحيد الإجباري هو ما بين BEGIN و END;
  • DECLARE و EXCEPTION كلاهما اختياري
  • يجب أن ينتهي الكود بـ END; مع فاصلة منقوطة
  • علامة / بعد END; تنفّذ الكتلة في SQL*Plus
  • كل جملة داخل الكتلة تنتهي بفاصلة منقوطة ;
