Creating DML Triggers

إنشاء المشغّلات

📖 ما هو Trigger؟ (Slide 2)

المشغّل (Trigger) هو كتلة PL/SQL أو إجراء مرتبط بجدول أو View أو Schema أو Database، ويُنفَّذ تلقائياً عند حدث معين.

النوع الشرح
Application Trigger يعمل عند حدث في تطبيق معين
Database Trigger يعمل عند حدث بيانات (DML) أو حدث نظام (Logon/Shutdown)

📋 إرشادات التصميم | Guidelines (Slide 3)

✅ استخدم Triggers لـ:
• تنفيذ إجراءات مرتبطة (Related Actions)
• تركيز العمليات العامة (Centralize Global Operations)

❌ لا تستخدم Triggers عندما:
• الوظيفة موجودة مسبقاً في Oracle Server
• تتكرر مع triggers أخرى
• الإفراط في الاستخدام يؤدي لتعقيدات صعبة الصيانة

⚙️ صيغة إنشاء DML Trigger (Slide 4)

CREATE [OR REPLACE] TRIGGER trigger_name
  timing
  event1 [OR event2 OR event3]
ON object_name
[[REFERENCING OLD AS old | NEW AS new]
  FOR EACH ROW
  [WHEN (condition)]]
trigger_body
العنصر الشرح
timing BEFORE | AFTER | INSTEAD OF
event INSERT | UPDATE [OF column] | DELETE
FOR EACH ROW يجعلها Row Trigger (بدونها = Statement Trigger)
WHEN شرط إضافي (اختياري)

🔄 أنواع DML Triggers (Slide 5)

النوع التنفيذ ملاحظات
Statement Trigger مرة واحدة لكل جملة DML النوع الافتراضي — يعمل حتى لو لم تتأثر أي صفوف
Row Trigger مرة لكل صف متأثر يُحدد بـ FOR EACH ROW — لا يعمل إذا لم تتأثر صفوف

⏰ توقيت التنفيذ | Trigger Timing (Slide 6)

التوقيت الشرح
BEFORE ينفذ قبل جملة DML على الجدول
AFTER ينفذ بعد جملة DML على الجدول
INSTEAD OF ينفذ بدلاً من الجملة — للـ Views غير القابلة للتعديل

⚠️ Note: إذا عُرّفت عدة triggers لنفس الكائن، ترتيب التنفيذ عشوائي (arbitrary).

📝 نقاط هامة | Key Takeaways

  • Trigger = كتلة PL/SQL تعمل تلقائياً عند حدث
  • الأنواع: Statement (مرة واحدة) vs Row (لكل صف)
  • التوقيت: BEFORE | AFTER | INSTEAD OF
  • الأحداث: INSERT | UPDATE [OF col] | DELETE
  • FOR EACH ROW = Row Trigger
