Triggers

المشغلات (Triggers)

🔫 ما هو المشغل (Trigger)؟

هو كتلة PL/SQL تُنفذ Automatically في قاعدة البيانات. لا يمكن استدعاؤه يدوياً مثل Procedure، بل "ينطلق" (Fires) عند حدوث شيء مثل INSERT أو UPDATE أو DELETE.

استخدامات شائعة: التدقيق الأمني، إنشاء سجلات مراقبة (Auditing)، التحقق من صحة البيانات المعقدة، النسخ الاحتياطي التلقائي.

⚙️ أنواع المشغلات

1. حسب مستوى التنفيذ (Timing Level)

  • Statement Trigger: ينطلق مرة واحدة للجملة ككل (حتى لو أثرت على 1000 صف). هذا هو الافتراضي.
  • Row Trigger: ينطلق مرة واحدة لكل صف يتأثر (نستخدم FOR EACH ROW).

2. حسب وقت التنفيذ (Timing Point)

  • BEFORE: قبل تنفيذ العملية (ممتاز للتحقق من البيانات).
  • AFTER: بعد تنفيذ العملية (ممتاز للتسجيل والـ Auditing).
  • INSTEAD OF: بدلاً من العملية (يستخدم مع الـ Views فقط).

💻 مثال: Row Trigger (تدقيق الرواتب)

هذا المثال يمنع تقليل راتب الموظف.

CREATE OR REPLACE TRIGGER check_salary
BEFORE UPDATE OF salary ON employees
FOR EACH ROW
BEGIN
    -- :OLD تشير للقيمة القديمة، :NEW تشير للقيمة الجديدة
    IF :NEW.salary < :OLD.salary THEN
        RAISE_APPLICATION_ERROR(-20001, 'لا يُسمح بتخفيض الراتب!');
    END IF;
END;
/

🔑 المتغيرات الخاصة :OLD و :NEW

تتوفر فقط في Row Triggers وتستخدم للوصول لقيم الأعمدة:

العملية :OLD :NEW
INSERT NULL القيمة المدخلة
UPDATE القيمة قبل التعديل القيمة بعد التعديل
DELETE القيمة قبل الحذف NULL
