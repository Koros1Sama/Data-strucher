Bind & Substitution Variables

متغيرات الربط والاستبدال

🤔 ما الفرق؟

هذان النوعان ليسا جزءاً من لغة PL/SQL نفسها، بل هما ميزات لأدوات التطوير (مثل SQL Developer, SQL*Plus) للتفاعل مع المستخدم.

الميزة متغيرات الاستبدال (&) متغيرات الربط (:)
الرمز & أو && : (مثل :variable)
الغرض تطلب قيمة من المستخدم قبل التنفيذ وتستبدلها في النص. متغير عام (Global) يُحفظ في البيئة ويمكن استخدامه بين كتل PL/SQL.
الاسم الشائع Substitution Variables Bind Variables (Host Variables)

1. متغيرات الاستبدال (Substitution Variables)

تستخدم لطلب مدخلات من المستخدم عند تشغيل الكود.

  • &variable: تسأل المستخدم في كل مرة تنفذ الكود.
  • &&variable: تسأل المستخدم مرة واحدة وتحفظ القيمة للجلسة الحالية. 
-- هذا الكود سيُظهر نافذة منبثقة تطلب إدخال رقم الموظف
DECLARE
    v_emp_id NUMBER := &emp_id; -- يتم استبدال &emp_id بالقيمة المدخلة
BEGIN
    DBMS_OUTPUT.PUT_LINE('Employee ID is: ' || v_emp_id);
END;

2. متغيرات الربط (Bind Variables)

يتم إنشاؤها باستخدام الأمر VARIABLE وتستخدم داخل كتل PL/SQL بوضع نقطتين : قبل الاسم. 

-- تعريف متغير ربط في البيئة (خارج PL/SQL)
VARIABLE g_salary NUMBER;

BEGIN
    -- استخدام المتغير بوضع : قبله
    SELECT salary INTO :g_salary 
    FROM employees 
    WHERE employee_id = 100;
END;
/

-- طباعة قيمة متغير الربط
PRINT g_salary;

⚠️ ملاحظة هامة

متغيرات الاستبدال (&) تتم معالجتها قبل إرسال الكود لقاعدة البيانات (كأنك قمت ببحث واستبدال في النص). أما متغيرات الربط (:) فهي variables حقيقية تُستخدم لتحسين الأداء والأمن.
