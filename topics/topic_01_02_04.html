Substitution Variables

متغيرات الاستبدال

📖 ما هي Substitution Variables؟

متغيرات الاستبدال:

  • تُستخدم للحصول على مدخلات من المستخدم أثناء التشغيل (at run time)
  • تُشار إليها بعلامة & (ampersand) قبل الاسم
  • تُستخدم لتجنب ترميز القيم الثابتة (hard coding) التي يمكن أخذها من المستخدم

💻 مثال من المنهج | Example (Slide 29)

VARIABLE emp_salary NUMBER
SET AUTOPRINT ON

DECLARE
  empno NUMBER(6) := &empno;  -- يطلب رقم الموظف من المستخدم
BEGIN
  SELECT salary INTO :emp_salary
  FROM   employees
  WHERE  employee_id = empno;
END;
/

💡 كيف يعمل: عند التنفيذ، يظهر للمستخدم مربع حوار (أو prompt) يطلب منه إدخال قيمة لـ &empno. يتم استبدال &empno بالقيمة المدخلة قبل التنفيذ.

⚖️ مقارنة: Bind vs Substitution

الخاصية Bind Variable Substitution Variable
الإنشاء VARIABLE name type &name داخل الكود
الإشارة :name (نقطتين) &name (ampersand)
وقت القيمة أثناء التنفيذ (Runtime) قبل التنفيذ (Pre-processing)
التخزين تبقى بعد انتهاء الكتلة تُستبدل وتختفي
الاستخدام تبادل بيانات بين الكتلة والبيئة إدخال مستخدم

📝 نقاط هامة | Key Takeaways

  • &variable = يُطلب من المستخدم إدخال قيمة
  • يتم الاستبدال قبل تنفيذ الكتلة
  • يختلف عن Bind Variable (:variable)
  • تُستخدم لتجنب hard coding القيم
