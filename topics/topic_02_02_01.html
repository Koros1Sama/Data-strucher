SQL Cursor & Implicit Cursor Attributes

خصائص المؤشر الضمني

📖 ما هو SQL Cursor؟ (Slide 35)

المؤشر (Cursor) هو مؤشر إلى منطقة ذاكرة خاصة يخصصها خادم Oracle. نوعان:

النوع الشرح
Implicit Cursors يُنشئها ويديرها Oracle تلقائياً لمعالجة جمل SQL
Explicit Cursors يُعرّفها ويديرها المبرمج صراحة

⚙️ خصائص المؤشر الضمني | Implicit Cursor Attributes

تُستخدم لاختبار نتيجة جمل SQL:

الخاصية الشرح
SQL%FOUND TRUE إذا أثرت/أرجعت آخر جملة على صف واحد على الأقل
SQL%NOTFOUND TRUE إذا لم تؤثر/ترجع آخر جملة على أي صف
SQL%ROWCOUNT عدد الصفوف المتأثرة/المُرجعة

مثال من المنهج (Slide 37)

VARIABLE rows_deleted VARCHAR2(30)
DECLARE
  empno employees.employee_id%TYPE := 176;
BEGIN
  DELETE FROM employees
  WHERE employee_id = empno;

  :rows_deleted := (SQL%ROWCOUNT || ' row deleted.');
END;
/
PRINT rows_deleted

📝 نقاط هامة | Key Takeaways

  • Implicit Cursor يُنشأ تلقائياً لكل جملة SQL
  • SQL%ROWCOUNT — عدد الصفوف المتأثرة (أكثر استخداماً)
  • SQL%FOUND / SQL%NOTFOUND — هل تأثر صف أم لا
  • تُستخدم مباشرة بعد جملة DML للتحقق من نتيجتها
