Composite Data Types & PL/SQL Records

أنواع البيانات المركبة والسجلات

📖 ما هي Composite Data Types؟

أنواع البيانات المركبة يمكنها حمل قيم متعددة بخلاف الأنواع القياسية (Scalar). نوعان رئيسيان:

النوع الاستخدام الأنواع الفرعية
PL/SQL Records تخزين قيم من أنواع بيانات مختلفة (حدوث واحد في المرة) User-defined Records, %ROWTYPE
PL/SQL Collections تخزين قيم من نفس نوع البيانات INDEX BY Tables, Nested Table, VARRAY

📋 PL/SQL Records (السجلات)

خصائص السجلات:

  • تحتوي على عنصر واحد أو أكثر (fields) من أنواع Scalar أو RECORD أو INDEX BY
  • مشابهة لـ Structures في C/C++
  • يعرّفها المستخدم ويمكن أن تمثل مجموعة فرعية من صف في جدول
  • مفيدة لجلب صف كامل من جدول للمعالجة

الصيغة (Slide 5)

-- الخطوة 1: تعريف النوع
TYPE type_name IS RECORD
     (field_declaration[, field_declaration]…);

-- field_declaration:
field_name {field_type | variable%TYPE
           | table.column%TYPE | table%ROWTYPE}
           [[NOT NULL] {:= | DEFAULT} expr]

-- الخطوة 2: إنشاء متغير من هذا النوع
identifier  type_name;

مثال (Slide 6)

DECLARE
  TYPE emp_record_type IS RECORD
    (last_name   VARCHAR2(25),
     job_id      VARCHAR2(10),
     salary      NUMBER(8,2));

  emp_record  emp_record_type;
BEGIN
  -- استخدام السجل
  emp_record.last_name := 'Ahmed';
  emp_record.salary := 5000;
END;

🔗 خاصية %ROWTYPE

تُعرّف متغيراً يأخذ جميع أعمدة جدول أو عرض (View) تلقائياً. 

DECLARE
  identifier  reference%ROWTYPE;

مزايا %ROWTYPE (Slide 9)

  • لا تحتاج لمعرفة عدد أو أنواع الأعمدة
  • تتكيّف تلقائياً إذا تغيّرت الأعمدة في الجدول
  • مفيدة جداً مع SELECT *

مثال: SELECT * مع %ROWTYPE (Slide 10)

DEFINE employee_number = 124
DECLARE
  emp_rec  employees%ROWTYPE;
BEGIN
  SELECT * INTO emp_rec FROM employees
  WHERE  employee_id = &employee_number;

  INSERT INTO retired_emps(empno, ename, job, mgr,
    hiredate, leavedate, sal, comm, deptno)
  VALUES (emp_rec.employee_id, emp_rec.last_name,
    emp_rec.job_id, emp_rec.manager_id,
    emp_rec.hire_date, SYSDATE, emp_rec.salary,
    emp_rec.commission_pct, emp_rec.department_id);
END;
/

مثال: INSERT باستخدام Record كاملاً (Slide 11)

DEFINE employee_number = 124
DECLARE
  emp_rec  retired_emps%ROWTYPE;
BEGIN
  SELECT employee_id, last_name, job_id, manager_id,
    hire_date, hire_date, salary, commission_pct,
    department_id INTO emp_rec FROM employees
  WHERE employee_id = &employee_number;

  INSERT INTO retired_emps VALUES emp_rec;  -- إدراج السجل كاملاً!
END;
/

مثال: UPDATE باستخدام SET ROW (Slide 12)

DEFINE employee_number = 124
DECLARE
  emp_rec retired_emps%ROWTYPE;
BEGIN
  SELECT * INTO emp_rec FROM retired_emps;
  emp_rec.leavedate := SYSDATE;

  UPDATE retired_emps SET ROW = emp_rec  -- تحديث الصف بالسجل كاملاً!
  WHERE empno = &employee_number;
END;
/

📝 نقاط هامة | Key Takeaways

  • Records = أنواع مختلفة | Collections = نفس النوع
  • TYPE name IS RECORD (...) لتعريف سجل
  • %ROWTYPE يأخذ جميع أعمدة الجدول تلقائياً
  • يمكن استخدام INSERT INTO table VALUES record
  • يمكن استخدام UPDATE table SET ROW = record
  • الوصول للحقول: record_name.field_name
