File Organizations & Buffering

تنظيم الملفات والتخزين المؤقت

📦 التخزين المؤقت | Buffering (Slides 121-123)

  • التخزين المؤقت مفيد عندما يمكن تشغيل العمليات بالتوازي
  • Double Buffering: يُستخدم لقراءة تدفق مستمر من الكتل

معلومات إدارة Buffer (Slide 123)

المعلومة الوصف
Pin Count عدد العمليات التي تستخدم الـ Buffer حالياً
Dirty Bit هل تم تعديل البيانات في الـ Buffer؟

استراتيجيات الاستبدال

  • LRU (Least Recently Used) — الأقل استخداماً مؤخراً
  • Clock Policy
  • FIFO (First-In-First-Out)

📋 Spanned vs Unspanned Records (Slides 124-125)

النوع الوصف
Spanned السجل يمكن أن يمتد عبر أكثر من كتلة — مؤشر في نهاية الكتلة يشير للجزء المتبقي
Unspanned السجل لا يتجاوز حدود الكتلة الواحدة

Blocking Factor = متوسط عدد السجلات في كل كتلة

📁 أنواع تنظيم الملفات (Slides 126-127)

النوع الإدراج البحث الحذف
Heap (Pile) File فعّال جداً (يُضاف في النهاية) بحث خطي (بطيء) إعادة كتابة أو علامة حذف
Ordered (Sequential) File معقد (يجب الحفاظ على الترتيب) Binary Search (سريع) فعّال نسبياً

📝 نقاط هامة | Key Takeaways

  • Double Buffering يُحسّن أداء القراءة المتتابعة
  • Pin Count + Dirty Bit = معلومات إدارة الـ Buffer
  • LRU: أشهر استراتيجية استبدال
  • Heap: إدراج سريع، بحث بطيء | Ordered: بحث سريع، إدراج بطيء
