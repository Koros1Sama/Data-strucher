Primary Index

الفهرس الأساسي

📖 التعريف (Slide 5)

  • يُعرّف على ملف بيانات مُرتّب (Ordered Data File)
  • ملف البيانات مُرتّب على حقل مفتاحي (Key Field) — قيم فريدة
  • يتضمن مدخل واحد لكل كتلة في ملف البيانات
  • المدخل يحتوي على قيمة المفتاح لأول سجل في الكتلة = Block Anchor
  • يُعتبر Nondense Index (فهرس غير كثيف)

📊 التمثيل البصري

Index File                Data File (Ordered by Key)
┌──────────────┐         ┌──────────────────┐
│ Key=5  → ────┼────────→│ Block 1: 5,8,12  │
├──────────────┤         ├──────────────────┤
│ Key=15 → ────┼────────→│ Block 2: 15,18,21│
├──────────────┤         ├──────────────────┤
│ Key=25 → ────┼────────→│ Block 3: 25,30,35│
└──────────────┘         └──────────────────┘

مدخل واحد لكل كتلة = Sparse (Nondense)

📝 نقاط هامة | Key Takeaways

  • ملف البيانات مُرتّب على حقل مفتاحي (قيم فريدة)
  • مدخل واحد لكل كتلة (ليس لكل سجل) → Sparse
  • Block Anchor = أول سجل في كل كتلة
  • البحث الثنائي على الفهرس ثم قراءة الكتلة المطلوبة
