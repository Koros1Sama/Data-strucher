Introduction to Transactions

مقدمة في المعاملات

📖 تعريف المعاملة | Transaction (Slides 2-3)

  • Transaction = برنامج قيد التنفيذ يُشكّل وحدة منطقية لمعالجة قاعدة البيانات
  • عبارات BEGIN و END TRANSACTION تحدد حدود المعاملة
  • أنواع المعاملات:
    • Read-Only Transaction: قراءة فقط
    • Read-Write Transaction: قراءة وكتابة

📊 أنظمة معالجة المعاملات

أنظمة بقواعد بيانات كبيرة ومئات المستخدمين المتزامنين تتطلب:

  • High Availability — توفر عالي
  • Fast Response Time — وقت استجابة سريع

🗃️ عناصر قاعدة البيانات | Database Items (Slide 4)

  • قاعدة البيانات = مجموعة من عناصر بيانات مُسمّاة
  • Granularity = حجم عنصر البيانات:
    • Record (سجل)
    • Disk Block (كتلة قرص)
    • Attribute Value (قيمة صفة)
  • مفاهيم المعاملات مستقلة عن حجم العنصر

📦 DBMS Buffers (Slide 5)

  • DBMS يحتفظ بعدة buffers في الذاكرة الرئيسية (Database Cache)
  • عند امتلاء الـ buffers → سياسة استبدال (مثل LRU)

📝 نقاط هامة | Key Takeaways

  • Transaction = وحدة منطقية لمعالجة قاعدة البيانات
  • Read-Only vs Read-Write transactions
  • Granularity: Record, Block, أو Attribute
  • DBMS يستخدم buffers + سياسة استبدال (LRU)
