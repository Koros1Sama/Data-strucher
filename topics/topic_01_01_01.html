Introduction to PL/SQL

مقدمة في لغة PL/SQL

📖 ما هي PL/SQL؟ | What Is PL/SQL?

PL/SQL هي اختصار لـ Procedural Language extension to SQL.

  • هي لغة Oracle القياسية للوصول إلى البيانات في قواعد البيانات العلائقية.
  • تدمج بسلاسة بين البنى الإجرائية (Procedural Constructs) ولغة SQL.

📋 خصائص PL/SQL | About PL/SQL

PL/SQL توفر:

  • بنية كتلية (Block Structure): للكود القابل للتنفيذ، مما يسهّل صيانة الكود.
  • بنى إجرائية مثل:
    • المتغيرات والثوابت والأنواع (Variables, Constants, Types)
    • هياكل التحكم مثل الشروط والحلقات (Conditional Statements & Loops)
    • وحدات برمجية قابلة لإعادة الاستخدام (Reusable Program Units)

🏗️ بيئة PL/SQL | PL/SQL Environment

عند إرسال كتلة PL/SQL إلى خادم Oracle، يتم تقسيمها بواسطة محرك PL/SQL (PL/SQL Engine): 

┌──────────────────────────────────┐
│        PL/SQL Block              │
│  ┌──────────┐ ┌───────────────┐  │
│  │ procedural│ │     SQL       │  │
│  │ statements│ │  statements   │  │
│  └─────┬─────┘ └──────┬────────┘  │
│        │              │           │
│        ▼              ▼           │
│  Procedural       SQL Statement  │
│  Statement        Executor       │
│  Executor    (Oracle DB Server)  │
└──────────────────────────────────┘

💡 كيف يعمل: الجمل الإجرائية (IF, LOOP ...) يعالجها محرك PL/SQL، أما جمل SQL (SELECT, INSERT ...) فتُرسل إلى منفّذ SQL في خادم Oracle.

⚡ مميزات PL/SQL | Benefits

الميزة الشرح
Integration with SQL تكامل البنى الإجرائية مع SQL — يمكنك كتابة IF...THEN مع جمل SQL في نفس الكتلة
Improved Performance إرسال كتلة كاملة من الأوامر للخادم دفعة واحدة بدلاً من إرسال كل جملة SQL على حدة، مما يقلل حركة الشبكة (Network Traffic)
Modularized Development تطوير معياري — تقسيم البرنامج إلى وحدات قابلة لإعادة الاستخدام
Integration with Oracle Tools تكامل مع أدوات Oracle المختلفة
Portability قابلية النقل — تعمل على أي نظام يدعم Oracle
Exception Handling معالجة الأخطاء والاستثناءات بشكل مدمج

📊 مقارنة الأداء (من المنهج):
بدون PL/SQL: كل جملة SQL تُرسل منفردة → رحلات شبكة متعددة
مع PL/SQL: كتلة كاملة تُرسل مرة واحدة → رحلة شبكة واحدة 

-- بدون PL/SQL (عدة رحلات شبكة):
SQL 1  →  Server
SQL 2  →  Server
SQL 3  →  Server

-- مع PL/SQL (رحلة واحدة):
IF...THEN
  SQL 1
ELSE
  SQL 2
END IF;
SQL 3
→  Server (مرة واحدة)

🖥️ بيئة التطوير و إخراج النتائج | Output

يمكن استخدام iSQL*Plus أو SQL*Plus أو SQL Developer كبيئة تطوير.

لطباعة النتائج، نستخدم:

-- أولاً: تفعيل الإخراج
SET SERVEROUTPUT ON

-- ثانياً: استخدام الحزمة المدمجة
DBMS_OUTPUT.PUT_LINE('The First Name of the Employee is ' || f_name);

⚠️ مهم: إذا لم تكتب SET SERVEROUTPUT ON أولاً، فلن ترى أي نتائج مطبوعة حتى لو استخدمت DBMS_OUTPUT.PUT_LINE.

📝 نقاط هامة | Key Takeaways

  • PL/SQL = Procedural Language extension to SQL
  • تُعالج الجمل الإجرائية في محرك PL/SQL وجمل SQL في خادم Oracle
  • تقلل حركة الشبكة بإرسال كتلة كاملة بدلاً من جمل منفردة
  • توفر: تطوير معياري، قابلية نقل، معالجة أخطاء، تكامل مع Oracle
  • SET SERVEROUTPUT ON + DBMS_OUTPUT.PUT_LINE لطباعة النتائج
