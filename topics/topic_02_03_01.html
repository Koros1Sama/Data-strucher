المؤشرات (Cursors)

📍 ما هو المؤشر (Cursor)؟

المؤشر هو Pointer يشير إلى منطقة خاصة في الذاكرة (Context Area) حيث يتم معالجة جملة SQL. يسمح لنا المؤشر بالتعامل مع نتيجة الاستعلام (SELECT) صفاً بصف (Row-by-Row).

🔄 أنواع المؤشرات

1. مؤشر ضمني (Implicit Cursor)

يتم إنشاؤه تلقائياً بواسطة Oracle عند تنفيذ أي جملة DML (INSERT, UPDATE, DELETE) أو جملة SELECT INTO (التي ترجع صفاً واحداً).

2. مؤشر صريح (Explicit Cursor)

يتم تعريفه يدوياً بواسطة المبرمج للتحكم في استعلام يرجع أكثر من صف (Multiple Rows).

📝 خطوات التعامل مع المؤشر الصريح (Explicit)

يجب اتباع 4 خطوات بالترتيب:

  1. DECLARE: تعريف المؤشر وتحديد جملة SELECT المرتبطة به.
  2. OPEN: فتح المؤشر (تنفيذ الاستعلام وحجز الذاكرة).
  3. FETCH: جلب البيانات صفاً تلو الآخر.
  4. CLOSE: إغلاق المؤشر وتحرير الذاكرة.

💻 مثال عملي

DECLARE
    -- 1. تعريف المؤشر
    CURSOR c_emps IS 
        SELECT last_name, salary FROM employees WHERE department_id = 90;
        
    v_name   employees.last_name%TYPE;
    v_sal    employees.salary%TYPE;
BEGIN
    -- 2. فتح المؤشر
    OPEN c_emps;
    
    LOOP
        -- 3. جلب صف واحد
        FETCH c_emps INTO v_name, v_sal;
        
        -- شرط الخروج: إذا لم نجد بيانات
        EXIT WHEN c_emps%NOTFOUND;
        
        DBMS_OUTPUT.PUT_LINE(v_name || ' earns ' || v_sal);
    END LOOP;
    
    -- 4. إغلاق المؤشر
    CLOSE c_emps;
END;

⚡ خصائص المؤشر (Cursor Attributes)

تستخدم لمعرفة حالة المؤشر (تسبق بـ %):

  • %FOUND: ترجع TRUE إذا تم جلب صف بنجاح.
  • %NOTFOUND: ترجع TRUE إذا لم يعد هناك صفوف.
  • %ROWCOUNT: عدد الصفوف التي تمت معالجتها حتى الآن.
  • %ISOPEN: هل المؤشر مفتوح أم لا؟
