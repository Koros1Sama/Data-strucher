Parameter Modes: IN, OUT, IN OUT

أنماط المعاملات: IN, OUT, IN OUT

📖 مقدمة | Introduction

أنماط المعاملات (Parameter Modes) تحدد اتجاه انتقال البيانات بين البيئة المُستدعية والبرنامج الفرعي. يوجد 3 أنماط وكل نمط له خصائص وقواعد محددة.

📊 جدول المقارنة الشامل | Comprehensive Comparison

الخاصية IN OUT IN OUT
الوضع الافتراضي؟ ✅ نعم (افتراضي) ❌ يجب تحديده ❌ يجب تحديده
الاتجاه → يمرر إلى البرنامج ← يُرجع من البرنامج ↔ يمرر ويُرجع
القراءة/الكتابة داخل البرنامج 📖 للقراءة فقط (Read-only) ✏️ للكتابة فقط (يجب التعيين) 📖✏️ قراءة وكتابة
القيمة الأولية تُمرر القيمة الفعلية تكون NULL عند الدخول تُمرر القيمة الفعلية
ما يمكن تمريره ثابت (Literal)، متغير، تعبير متغير فقط (Variable) متغير فقط (Variable)
القيمة الافتراضية ✅ يمكن استخدام DEFAULT أو := ❌ لا يمكن ❌ لا يمكن

📥 معامل IN | IN Parameter

🔑 القاعدة: IN هو الوضع الافتراضي. يمرر قيمة للبرنامج الفرعي ولا يمكن تعديل قيمته داخله. يشبه الثابت (CONSTANT). 

CREATE OR REPLACE PROCEDURE raise_salary
  (id      IN employees.employee_id%TYPE,   -- IN (صريح)
   percent    NUMBER)                        -- IN (ضمني - افتراضي)
IS
BEGIN
  UPDATE employees
  SET salary = salary * (1 + percent/100)
  WHERE employee_id = id;
END raise_salary;
/

-- الاستدعاء:
EXECUTE raise_salary(176, 10);

-- يمكن تمرير ثابت أو متغير أو تعبير:
EXECUTE raise_salary(v_id, 5 + 5);

⚠️ خطأ شائع: لا يمكنك تغيير قيمة معامل IN داخل الإجراء: 

CREATE PROCEDURE test(x IN NUMBER) IS
BEGIN
  x := 10; -- ❌ خطأ! PLS-00363: expression 'X' cannot be used as an assignment target
END;

📤 معامل OUT | OUT Parameter

🔑 القاعدة: OUT يُرجع قيمة من البرنامج الفرعي إلى البيئة المُستدعية. القيمة الأولية NULL ويجب تعيين قيمة له قبل انتهاء البرنامج. 

CREATE OR REPLACE PROCEDURE query_emp
  (id     IN  employees.employee_id%TYPE,
   name   OUT employees.last_name%TYPE,
   salary OUT employees.salary%TYPE,
   comm   OUT employees.commission_pct%TYPE)
IS
BEGIN
  SELECT last_name, salary, commission_pct
  INTO   name, salary, comm
  FROM   employees
  WHERE  employee_id = id;
END query_emp;
/

استدعاء إجراء يحتوي OUT من بلوك PL/SQL

SET SERVEROUTPUT ON
DECLARE
  emp_name  employees.last_name%TYPE;
  emp_sal   employees.salary%TYPE;
  emp_comm  employees.commission_pct%TYPE;
BEGIN
  query_emp(171, emp_name, emp_sal, emp_comm);
  DBMS_OUTPUT.PUT_LINE('Name: '    || emp_name);
  DBMS_OUTPUT.PUT_LINE('Salary: '  || emp_sal);
  DBMS_OUTPUT.PUT_LINE('Comm: '    || NVL(TO_CHAR(emp_comm), 'NULL'));
END;
/

استدعاء باستخدام Host Variables

VARIABLE name  VARCHAR2(25)
VARIABLE sal   NUMBER
VARIABLE comm  NUMBER

EXECUTE query_emp(171, :name, :sal, :comm)
PRINT name sal comm

🔄 معامل IN OUT | IN OUT Parameter

🔑 القاعدة: IN OUT يمرر قيمة أولية للبرنامج الفرعي ويتم تعديلها وإعادتها. يجمع بين خصائص IN و OUT. 

CREATE OR REPLACE PROCEDURE format_phone
  (phone_no IN OUT VARCHAR2)
IS
BEGIN
  phone_no := '(' || SUBSTR(phone_no, 1, 3) || ') '
              || SUBSTR(phone_no, 4, 3) || '-'
              || SUBSTR(phone_no, 7);
END format_phone;
/

-- الاستدعاء:
DECLARE
  phone_no VARCHAR2(13) := '8006330575';
BEGIN
  format_phone(phone_no);
  DBMS_OUTPUT.PUT_LINE('Formatted: ' || phone_no);
  -- Output: Formatted: (800) 633-0575
END;
/

💡 لاحظ: المتغير phone_no دخل بقيمة '8006330575' وخرج بقيمة '(800) 633-0575'. هذا جوهر IN OUT — يستقبل ويُعيد.

📝 نقاط هامة | Key Takeaways

  • IN = افتراضي، للقراءة فقط، يقبل ثوابت ومتغيرات وتعابير
  • OUT = يُرجع قيمة، يبدأ NULL، يقبل متغيرات فقط
  • IN OUT = يستقبل ويُرجع، يقبل متغيرات فقط
  • IN فقط يدعم القيم الافتراضية (DEFAULT / :=)
  • OUT و IN OUT لا يمكنهما استخدام DEFAULT
  • لا يمكن تعيين قيمة لمعامل IN (يعامل كثابت)
