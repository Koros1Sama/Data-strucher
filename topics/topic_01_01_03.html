Block Types

أنواع كتل PL/SQL

🧩 الأنواع الثلاثة | The Three Types

يمكن تقسيم كتل PL/SQL إلى ثلاثة أنواع رئيسية:

1. كتلة مجهولة (Anonymous Block)

ليس لها اسم، لا تُحفظ في قاعدة البيانات. تُنفذ مرة واحدة فقط ثم تختفي.

للتجربة السريعة والاختبار

2. إجراء (Procedure)

لها اسم، تُحفظ في قاعدة البيانات كـ schema object. يمكن استدعاؤها مرارًا. تستخدم لتنفيذ فعل (Action).

برنامج فرعي محفوظ

3. دالة (Function)

مثل الإجراء لكنها يجب أن تُرجع قيمة باستخدام RETURN. تُستخدم عادة للحسابات.

ترجع قيمة إجبارياً

💻 الصيغ من المنهج | Syntax (Slide 9)

Anonymous Block

[DECLARE]

BEGIN
  --statements

[EXCEPTION]

END;

Procedure

PROCEDURE name
IS

BEGIN
  --statements

[EXCEPTION]

END;

Function

FUNCTION name
RETURN datatype
IS
BEGIN
  --statements
  RETURN value;
[EXCEPTION]

END;

⚖️ مقارنة سريعة | Comparison

الخاصية Anonymous Procedure Function
الاسم ❌ لا يوجد ✅ موجود ✅ موجود
الحفظ في DB ❌ لا ✅ نعم ✅ نعم
RETURN قيمة ❌ لا اختياري (عبر OUT) ✅ إجباري
يبدأ بـ DECLARE أو BEGIN PROCEDURE name IS FUNCTION name RETURN type IS
إعادة الاستخدام ❌ مرة واحدة ✅ عدة مرات ✅ عدة مرات

📝 نقاط هامة | Key Takeaways

  • Anonymous Block: بدون اسم، لا تُحفظ، للتنفيذ المباشر
  • Procedure: لها اسم، تُحفظ، تنفذ فعل (لا ترجع قيمة رسمياً)
  • Function: لها اسم، تُحفظ، يجب أن ترجع قيمة بـ RETURN
  • Procedure و Function = Subprograms (برامج فرعية)
