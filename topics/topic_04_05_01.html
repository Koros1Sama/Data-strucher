Disk Storage & File Structures

التخزين على القرص وبنى الملفات

📖 هرمية التخزين (Slide 111)

المستوى الأمثلة السرعة
Primary Storage CPU Cache, Main Memory (RAM) الأسرع
Secondary Storage Magnetic Disks, Flash/SSD متوسط
Tertiary Storage Removable Media (Tapes) الأبطأ

💿 تركيب القرص (Slides 112-113)

  • Persistent Data: بيانات تبقى بعد إنهاء البرنامج (معظم قواعد البيانات)
  • Transient Data: بيانات موجودة فقط أثناء التنفيذ
  • Tracks (مسارات): دوائر متحدة المركز على القرص
  • Blocks/Sectors: أقسام المسار — وحدة نقل البيانات
  • Cylinder: نفس المسار على جميع أسطح القرص

⚙️ الوصول للبيانات (Slides 116-118)

  • Formatting: تقسيم المسارات إلى كتل متساوية مع فجوات بينها (Interblock Gaps)
  • Buffer: منطقة في الذاكرة لعمليات القراءة والكتابة
  • Disk Controller: واجهة بين القرص والحاسب (SCSI, SATA, SAS)

💡 تقنيات الوصول الفعّال:
• Data Buffering
• ترتيب البيانات على القرص
• قراءة البيانات مسبقاً (Read Ahead)
• جدولة طلبات I/O
• استخدام Log Disks للكتابة المؤقتة
• استخدام SSD/Flash للاسترداد

💾 SSD (Slide 119)

  • يُسمى أيضاً Flash Storage
  • لا توجد أجزاء متحركة
  • البيانات أقل عرضة للتجزئة (Fragmentation)
  • أسرع في الوصول لكن أغلى ثمناً من HDD
  • DRAM-based SSDs: وصول أسرع من Flash

📝 نقاط هامة | Key Takeaways

  • 3 مستويات: Primary → Secondary → Tertiary
  • القرص: Tracks → Blocks/Sectors | Cylinder = نفس المسار عبر الأسطح
  • نقل البيانات بوحدات Disk Blocks
  • SSD: بدون أجزاء متحركة، أسرع، أغلى
