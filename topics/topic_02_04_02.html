Cursor FOR Loops

حلقات FOR للمؤشرات

📖 المفهوم | What is it? (Slide 72)

Cursor FOR Loop هو اختصار لمعالجة المؤشرات الصريحة. يقوم تلقائياً بـ:

  • ✅ OPEN ضمنياً
  • ✅ FETCH ضمنياً
  • ✅ EXIT ضمنياً عند انتهاء الصفوف
  • ✅ CLOSE ضمنياً
  • ✅ السجل (Record) يُعرّف ضمنياً
-- الصيغة:
FOR record_name IN cursor_name LOOP
  statement1;
  statement2;
  ...
END LOOP;

💻 مثال من المنهج (Slide 73)

SET SERVEROUTPUT ON
DECLARE
  CURSOR emp_cursor IS
    SELECT employee_id, last_name FROM employees
    WHERE department_id = 30;
BEGIN
  FOR emp_record IN emp_cursor LOOP
    DBMS_OUTPUT.PUT_LINE(emp_record.employee_id
      || ' ' || emp_record.last_name);
  END LOOP;
  -- لا حاجة لـ OPEN أو FETCH أو CLOSE!
END;
/

🔍 Cursor FOR Loop باستخدام Subquery (Slide 77)

لا حاجة حتى لتعريف المؤشر — ضع الاستعلام مباشرة:

SET SERVEROUTPUT ON
BEGIN
  FOR emp_record IN (SELECT employee_id, last_name
    FROM employees WHERE department_id = 30)
  LOOP
    DBMS_OUTPUT.PUT_LINE(emp_record.employee_id
      || ' ' || emp_record.last_name);
  END LOOP;
END;
/

💡 الفرق: في هذه الحالة لا نحتاج قسم DECLARE نهائياً!

⚖️ مقارنة: الطرق الثلاث

الطريقة OPEN/FETCH/CLOSE DECLARE cursor
Manual (LOOP + FETCH) يدوي كامل ✅ نعم
FOR Loop (مع Cursor) تلقائي ✅ نعم
FOR Loop (مع Subquery) تلقائي ❌ لا حاجة

📝 نقاط هامة | Key Takeaways

  • FOR Loop = اختصار: OPEN + FETCH + EXIT + CLOSE تلقائياً
  • السجل (record) يُعرّف تلقائياً
  • يمكن استخدام Subquery بدل Cursor المعرّف
  • الأكثر شيوعاً واستخداماً في الممارسة العملية
