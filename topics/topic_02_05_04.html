Procedures vs Functions

الفرق بين الإجراءات والدوال

📖 نظرة عامة | Overview

كلاً من الإجراءات (Procedures) والدوال (Functions) هي برامج فرعية مُسمّاة (Named Subprograms) في PL/SQL. يتم تخزينها في قاعدة البيانات وتُعزز إعادة الاستخدام والصيانة. لكنهما يختلفان في عدة نقاط جوهرية.

📊 جدول المقارنة الشامل | Full Comparison

وجه المقارنة الإجراء (Procedure) الدالة (Function)
طريقة التنفيذ يُنفذ كجملة PL/SQL مستقلة يُستدعى كجزء من تعبير (Expression)
RETURN في الرأس ❌ لا يحتوي على جملة RETURN في الرأس ✅ يجب أن يحتوي على RETURN datatype
القيمة المرجعة يمكن إرجاع قيم عبر معاملات OUT (اختياري) يجب أن يُرجع قيمة واحدة
جملة RETURN في الجسم يمكن أن يحتوي RETURN بدون قيمة (للخروج المبكر) يجب أن يحتوي جملة RETURN واحدة على الأقل
الاستخدام في SQL ❌ لا يمكن استدعاؤه في جمل SQL ✅ يمكن استخدامه في SELECT, WHERE, INSERT, UPDATE
أنماط المعاملات يدعم IN, OUT, IN OUT يقبل IN فقط عند استدعائه من SQL
الغرض الرئيسي تنفيذ عملية (Action) مثل تعديل بيانات حساب قيمة (Computation) وإرجاعها
الاستدعاء EXECUTE proc_name(...); var := func_name(...);
الحذف DROP PROCEDURE name; DROP FUNCTION name;
عرض الكود type = 'PROCEDURE' في USER_SOURCE type = 'FUNCTION' في USER_SOURCE

💻 مقارنة بالكود | Code Comparison

إجراء: رفع الراتب (Procedure)

CREATE OR REPLACE PROCEDURE raise_salary
  (id IN employees.employee_id%TYPE,
   percent IN NUMBER)
IS
BEGIN
  UPDATE employees
  SET salary = salary * (1 + percent/100)
  WHERE employee_id = id;
END raise_salary;
/

-- الاستدعاء:
EXECUTE raise_salary(176, 10);

دالة: الحصول على الراتب (Function)

CREATE OR REPLACE FUNCTION get_sal
  (id employees.employee_id%TYPE)
RETURN NUMBER IS
  sal employees.salary%TYPE := 0;
BEGIN
  SELECT salary INTO sal
  FROM employees
  WHERE employee_id = id;
  RETURN sal;
END get_sal;
/

-- الاستدعاء:
SELECT get_sal(100) FROM dual;
-- أو:
VARIABLE salary NUMBER
EXECUTE :salary := get_sal(100)

🤔 متى نستخدم أيهما؟ | When to Use Which?

استخدم الإجراء (Procedure) عندما:
• تريد تنفيذ عملية DML (INSERT, UPDATE, DELETE)
• تحتاج إرجاع أكثر من قيمة (عبر OUT parameters)
• لا تحتاج استدعاءه من جملة SQL

استخدم الدالة (Function) عندما:
• تحتاج حساب قيمة واحدة وإرجاعها
• تريد استخدامها داخل جمل SQL (SELECT, WHERE, etc.)
• تحتاج تعبيراً يمكن دمجه في حسابات أخرى

📝 نقاط هامة للاختبار | Key Exam Points

  • الإجراء = ينفذ عملية | الدالة = تحسب قيمة
  • الدالة يجب أن تحتوي RETURN في الرأس والجسم
  • الإجراء لا يحتوي RETURN datatype في الرأس
  • الدالة يمكن استدعاؤها في SELECT، الإجراء لا
  • الدوال من SQL لا تحتوي DML ولا COMMIT/ROLLBACK
  • كلاهما يُخزن في USER_SOURCE ويُحذف بـ DROP
