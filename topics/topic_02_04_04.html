Cursors with Parameters

المؤشرات ذات المعاملات

⚠️ تنبيه: هذا الموضوع قد لا يكون ضمن المنهج المقرر. لم يرد ذكره في سلايدات المحاضرات. يُنصح بمراجعته كمعلومات إضافية فقط وليس كمادة اختبارية.

📖 مقدمة | Introduction

يمكن تعريف المؤشر مع معاملات (Parameters) لجعله أكثر مرونة وإعادة استخداماً. بدلاً من ربط المؤشر بقيم ثابتة، نُمرر القيم عند فتحه.

💡 الفائدة: نفس المؤشر يُفتح بقيم مختلفة في كل مرة — مثل الإجراء المخزن لكن للاستعلامات.

⚙️ الصيغة العامة | Syntax

CURSOR cursor_name
  (parameter_name datatype, ...)
IS
  select_statement;

⚠️ ملاحظات:
• المعاملات لا تحدد حجمها (مثلاً NUMBER وليس NUMBER(10))
• المعاملات تكون دائماً IN (للقراءة فقط)
• يمكن وضع قيمة افتراضية بـ DEFAULT

💻 مثال عملي | Example

تعريف المؤشر بمعامل

DECLARE
  CURSOR emp_cursor(deptno NUMBER) IS
    SELECT employee_id, last_name
    FROM employees
    WHERE department_id = deptno;
BEGIN
  -- فتح المؤشر لقسم 10
  OPEN emp_cursor(10);
  -- ...
  CLOSE emp_cursor;

  -- إعادة فتحه لقسم 20
  OPEN emp_cursor(20);
  -- ...
  CLOSE emp_cursor;
END;
/

مع Cursor FOR Loop

SET SERVEROUTPUT ON
DECLARE
  CURSOR emp_cursor(deptno NUMBER) IS
    SELECT employee_id, last_name, salary
    FROM employees
    WHERE department_id = deptno;
BEGIN
  DBMS_OUTPUT.PUT_LINE('--- Department 10 ---');
  FOR emp_rec IN emp_cursor(10) LOOP
    DBMS_OUTPUT.PUT_LINE(
      emp_rec.employee_id || ' ' ||
      emp_rec.last_name || ' $' ||
      emp_rec.salary);
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('--- Department 20 ---');
  FOR emp_rec IN emp_cursor(20) LOOP
    DBMS_OUTPUT.PUT_LINE(
      emp_rec.employee_id || ' ' ||
      emp_rec.last_name || ' $' ||
      emp_rec.salary);
  END LOOP;
END;
/

🔣 معاملات متعددة وقيم افتراضية | Multiple & Default

DECLARE
  CURSOR emp_cursor(
    p_deptno NUMBER,
    p_job    VARCHAR2 DEFAULT 'ST_CLERK')
  IS
    SELECT employee_id, last_name, salary
    FROM employees
    WHERE department_id = p_deptno
      AND job_id = p_job;
BEGIN
  -- استخدام كلا المعاملين:
  FOR rec IN emp_cursor(50, 'ST_MAN') LOOP
    DBMS_OUTPUT.PUT_LINE(rec.last_name);
  END LOOP;

  -- استخدام القيمة الافتراضية لـ p_job:
  FOR rec IN emp_cursor(50) LOOP
    DBMS_OUTPUT.PUT_LINE(rec.last_name || ' (default job)');
  END LOOP;
END;
/

📝 نقاط هامة | Key Takeaways

  • المعاملات تجعل المؤشر قابلاً لإعادة الاستخدام
  • لا تحدد حجم النوع (VARCHAR2 وليس VARCHAR2(50))
  • المعاملات دائماً IN (للإدخال فقط)
  • تُمرر القيم عند OPEN أو في FOR cursor_name(value)
  • يمكن فتح نفس المؤشر عدة مرات بقيم مختلفة
  • يمكن استخدام DEFAULT لقيم افتراضية
