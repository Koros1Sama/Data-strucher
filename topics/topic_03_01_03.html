Using OLD and NEW Qualifiers

استخدام OLD و NEW في المشغّلات

📖 المفهوم | Concept

في Row Triggers يمكنك الوصول إلى القيم القديمة والجديدة للصف:

المرجع الاستخدام متاح في
:OLD.column القيمة قبل التعديل UPDATE, DELETE
:NEW.column القيمة بعد التعديل (أو المُدرجة) INSERT, UPDATE

⚠️ ملاحظة: :OLD تساوي NULL في INSERT، و:NEW تساوي NULL في DELETE. في WHEN clause تُستخدم بدون : (بدون نقطتين).

💻 مثال: Audit Trail (Slide 14)

CREATE OR REPLACE TRIGGER audit_emp_values
AFTER DELETE OR INSERT OR UPDATE ON employees
FOR EACH ROW
BEGIN
  INSERT INTO audit_emp(user_name, time_stamp, id,
    old_last_name, new_last_name, old_title,
    new_title, old_salary, new_salary)
  VALUES (USER, SYSDATE, :OLD.employee_id,
    :OLD.last_name, :NEW.last_name, :OLD.job_id,
    :NEW.job_id, :OLD.salary, :NEW.salary);
END;
/

🧪 اختبار audit_emp (Slide 15)

-- إدراج:
INSERT INTO employees
  (employee_id, last_name, job_id, salary, ...)
VALUES (999, 'Temp emp', 'SA_REP', 6000, ...);

-- تحديث:
UPDATE employees
  SET salary = 7000, last_name = 'Smith'
  WHERE employee_id = 999;

-- التحقق من جدول Audit:
SELECT user_name, timestamp, ...
FROM audit_emp;

🎯 تقييد Row Trigger بشرط WHEN (Slide 16)

CREATE OR REPLACE TRIGGER derive_commission_pct
BEFORE INSERT OR UPDATE OF salary ON employees
FOR EACH ROW
WHEN (NEW.job_id = 'SA_REP')   -- بدون : هنا!
BEGIN
  IF INSERTING THEN
    :NEW.commission_pct := 0;
  ELSIF :OLD.commission_pct IS NULL THEN
    :NEW.commission_pct := 0;
  ELSE
    :NEW.commission_pct := :OLD.commission_pct + 0.05;
  END IF;
END;
/

📌 فرق مهم:
• في WHEN clause: NEW.column (بدون :)
• في trigger_body: :NEW.column (مع :)

📝 نقاط هامة | Key Takeaways

  • :OLD = القيمة قبل التغيير | :NEW = القيمة الجديدة
  • INSERT: :OLD = NULL | DELETE: :NEW = NULL
  • WHEN clause: بدون : | trigger body: مع :
  • يمكن تعديل :NEW في BEFORE triggers
