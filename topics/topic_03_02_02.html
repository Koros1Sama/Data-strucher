Conditional Predicates: INSERTING, UPDATING, DELETING

المحمولات الشرطية: الإضافة، التحديث، الحذف

📖 مقدمة | Introduction

عند إنشاء مشغل واحد يتعامل مع عدة أحداث (مثلاً INSERT OR UPDATE OR DELETE)، نحتاج لطريقة لمعرفة أي حدث تسبب في تشغيل هذا المشغل. هنا يأتي دور المحمولات الشرطية (Conditional Predicates).

📝 المحتوى | Content

المحمولات الثلاثة

  • INSERTING: تعود بـ TRUE إذا كانت العملية هي إضافة (INSERT).
  • UPDATING: تعود بـ TRUE إذا كانت العملية هي تحديث (UPDATE).
  • DELETING: تعود بـ TRUE إذا كانت العملية هي حذف (DELETE).

استخدام UPDATING مع عمود محدد

يمكن التحقق مما إذا كان تحديث عمود معين هو السبب:

IF UPDATING('SALARY') THEN ...

مثال شامل

CREATE OR REPLACE TRIGGER audit_trigger
AFTER INSERT OR UPDATE OR DELETE ON employees
BEGIN
  IF INSERTING THEN
    INSERT INTO log_table VALUES ('New Employee Added');
  ELSIF DELETING THEN
    INSERT INTO log_table VALUES ('Employee Deleted');
  ELSIF UPDATING('SALARY') THEN
    INSERT INTO log_table VALUES ('Salary Updated');
  ELSE
    INSERT INTO log_table VALUES ('Other Update');
  END IF;
END;
