Trigger Firing Sequence

ترتيب تنفيذ المشغّلات

📖 ترتيب التنفيذ لصف واحد (Slide 7)

عند تعديل صف واحد:

  1. BEFORE Statement trigger
  2. BEFORE Row trigger
  3. AFTER Row trigger
  4. AFTER Statement trigger
-- مثال: إدراج صف واحد
INSERT INTO departments
  (department_id, department_name, location_id)
VALUES (400, 'CONSULTING', 2400);

📖 ترتيب التنفيذ لعدة صفوف (Slide 8)

عند تعديل عدة صفوف:

  1. BEFORE Statement trigger (مرة واحدة)
  2. لكل صف:
    • BEFORE Row trigger
    • AFTER Row trigger
  3. AFTER Statement trigger (مرة واحدة)
-- مثال: تحديث عدة صفوف
UPDATE employees
  SET salary = salary * 1.1
  WHERE department_id = 30;
-- سيُنفّذ Row Trigger مرة لكل موظف في القسم 30

⚡ ملخص نموذج التنفيذ | Execution Model (Slide 17)

الخطوة التفاصيل
1 Execute all BEFORE STATEMENT triggers
2 Loop for each row affected:
a. Execute all BEFORE ROW triggers
b. Execute DML + integrity constraint checking
c. Execute all AFTER ROW triggers
3 Execute all AFTER STATEMENT triggers

📌 Note: Integrity checking يمكن تأجيله حتى عملية COMMIT.

📝 نقاط هامة | Key Takeaways

  • Statement triggers: تعمل مرة في البداية ومرة في النهاية
  • Row triggers: تتكرر لكل صف متأثر
  • الترتيب: BEFORE Statement → (BEFORE Row → DML → AFTER Row) × N → AFTER Statement
  • DML يُنفّذ بين BEFORE ROW و AFTER ROW
